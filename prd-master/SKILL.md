# PRD MASTER（顶级产品需求文档专家）

本技能用于生成 **企业级完整 PRD 文档**，覆盖从产品战略、业务流程、信息架构、功能设计、数据结构、接口规范到运营规则的完整体系。

输出文档可直接用于：

• 产品评审  
• 研发开发  
• 测试用例设计  
• UI设计  
• 项目管理

# 使用指南

## 生成完整 PRD

当用户要求：生成XX系统的 PRD 文档时，按照以下步骤：

1. **收集需求信息**: 询问用户项目的基本信息
   - 项目名称和背景
   - 主要功能模块
   - 目标用户群体
   - 技术要求

2. **系统输出**：完整PRD文档，包括：

  - 信息架构  
  - 业务流程  
  - 功能设计  
  - 数据模型  
  - API接口  
  - 交互规范

3. **提供输出格式**: 支持 Markdown 格式输出

## 生成特定模块

当用户要求生成特定功能模块的 PRD 时：

1. **确认模块名称**: 明确用户需要哪个功能模块的详细说明

2. **生成模块内容**: 包含以下内容：
   - 功能描述
   - 核心功能列表
   - 用户场景
   - 页面设计
   - 数据结构

3. **提供示例**: 如有必要，提供相关的示例数据

## 优化现有 PRD

当用户提供现有 PRD 并要求优化时：

1. **分析现有文档**: 检查文档的完整性和规范性

2. **识别问题**: 找出缺失的内容或不规范的地方

3. **提供改进建议**: 给出具体的改进建议和修改方案
